2. **Task - Backend CPR filter implementation** (v0.1.3): Implemented server-side CPR escape sequence filtering via TermSocket monkey-patch<br>
   **Result**: Created `handlers.py` with `CPRFilteredTermSocket` class that subclasses `jupyter_server_terminals.handlers.TermSocket` and overrides `on_pty_read()` to filter CPR (`\x1b[\d+;\d+R`) and DA (`\x1b[\?[\d;]*c`) escape sequences before they reach the frontend. Filter function returns counts of intercepted sequences and logs them via Python logging module for debugging. Updated `__init__.py` to monkey-patch `TermSocket` during extension load - this intercepts terminal buffer drains during client reconnection. Simplified `src/index.ts` to remove API call boilerplate since filtering happens server-side. Deleted obsolete files: `routes.py`, `request.ts`, `test_routes.py`. Created `test_handlers.py` with 11 unit tests covering single/multiple CPR sequences, DA sequences, mixed content, preserved escape codes (colors, cursor movement), edge cases. Added `skipLibCheck: true` to `tsconfig.json` to resolve lib0 TypeScript compatibility issue. All tests pass.

5. **Task - Expand filter and fix monkey-patch** (v0.9.15): Extended filter to cover all terminal query response types and fixed critical monkey-patch bug<br>
   **Result**: Observed real fish shell output showing sequences not covered by original filter: OSC color responses (`ESC]10;rgb:...`, `ESC]11;rgb:...`), DA2 (`ESC[>...c`), and DECRPM (`ESC[?mode;value$y`). 6. **Task - Fix bare sequence filtering** (v0.9.18): Root-caused and fixed why CPR sequences leaked through the filter<br>
   **Result**: Built PTY simulation scripts (`test_pty_raw.py`) to reproduce the issue in isolation. Discovered that fish shell strips the ESC byte (`\x1b`) from terminal query responses and echoes the remainder as bare text - so `\x1b[2;1R` becomes `[2;1R` in PTY output. The regex `\x1b\[\d+;\d+R` requires the ESC prefix and never matched. Added 4 bare-pattern regexes with negative lookbehind `(?<!\x1b)` for CPR, DA, DA2, and DECRPM to filter sequences both with and without ESC prefix. Normal bracket usage (`array[0]`) is preserved because bare patterns require terminal-specific suffixes (`;digitR`, `;digitc`, `$y`). All 21 unit tests pass including tests for actual fish shell PTY output patterns.

7. **Task - OSC 52 exemption and diagnostics** (v0.9.19): Exempt OSC 52 clipboard sequences from filter and add diagnostic logging for bare pattern false positives<br>
   **Result**: Applied negative lookahead `(?!52;)` to the `ESC_OSC` regex in `handlers.py` so OSC 52 clipboard sequences pass through to the frontend - required by the companion `jupyterlab_terminal_clipboard_extension`. Added diagnostic logging in `__init__.py` that shows matched text with surrounding context for bare patterns, enabling investigation of potential false positives in colored terminal output. Log analysis showed bare_cpr/bare_da patterns matching in normal 4095-byte colored output chunks, confirming the patterns need tightening in a future release.
